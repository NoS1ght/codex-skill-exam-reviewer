第三章：关联规则挖掘

支持度/置信度 · Apriori算法 · FP-Growth · 模式评估

核心知识点总结

一、关联规则的基本概念

概念定义公式
支持度计数包含项集的事务数sc(X) = 包含X的事务数
支持度项集出现的频率support(X) = sc(X) / 总事务数
置信度规则X→Y的条件概率confidence(X→Y) = support(X∪Y) / support(X)
强关联规则：同时满足最小支持度阈值和最小置信度阈值的规则。
✦ 支持度：筛掉出现太少的项集（无代表性）
✦ 置信度：筛掉关心不强的规则（可信度低）

二、挖掘关联规则的一般步骤

  1. 找出所有频繁项集（满足最小支持度）
  2. 从频繁项集中产生强关联规则（满足最小置信度）

三、Apriori 算法

两个核心性质（先验原理）

性质1：频繁项集的所有非空子集也必须是频繁的。
性质2（逆反）：如果一个项集是非频繁的，则它的所有超集也一定是非频繁的。
→ 剪枝依据：任何非频繁(k-1)-项集都不是频繁k-项集的子集。

Apriori 算法流程

  • ① 扫描数据库，统计每个项的计数，找出频繁 1-项集 L₁
  • ② 由 L₁ 产生候选 2-项集 C₂，扫描数据库计算支持度，筛选得 L₂
  • ③ 由 L₂ 产生 C₃ → 扫描筛选 → L₃，以此类推
  • ④ 重复直到没有新的频繁项集产生
  • ⑤ 从频繁项集中生成关联规则，计算置信度并筛选
重要细节：连接步与剪枝步
✦ 连接步：L_{k-1} 自连接产生 C_k（前 k-2 项相同才连接）
✦ 剪枝步：利用先验性质，删除 C_k 中所有 (k-1)-子集不在 L_{k-1} 中的候选

四、FP-Growth 算法

  • 优点：不产生候选集，比Apriori快，只需两次扫描
  • 流程：① 第一次扫描：找频繁1-项集 → ② 第二次扫描：构建 FP 树 → ③ 递归挖掘FP树
  • FP树：压缩的、前缀树结构，存储频繁模式的完整信息

五、关联模式的评估

度量含义公式
支持度规则覆盖的事务比例P(X∪Y)
置信度规则的可靠性P(Y|X)
提升度规则的相关性（>1正相关，<1负相关，=1独立）P(X∪Y)/(P(X)P(Y))
记忆口诀：Apriori 核心思想
✦ 子集频繁（频繁→子集也频繁）
✦ 超集不频繁（不频繁→超集也不频繁）
✦ 层层递进：k-项集 → 候选(k+1)-项集 → 筛选 → …

互动自测

—— 选择题 ——

Q1关联规则 X→Y 的置信度计算公式为（ ）。单选
A. support(X∪Y) / 总事务数
B. support(Y) / support(X)
C. support(X∪Y) / support(X)
D. support(X∪Y) × support(X)
✓ C
解析：置信度 = P(Y|X) = P(X∪Y) / P(X) = support(X∪Y)/support(X)。A是支持度的公式，B和D不成立。
Q2Apriori 算法的先验性质表明：如果某 k-项集是频繁的，则它的所有 (k-1)-子集（ ）。单选
A. 一定是非频繁的
B. 一定是频繁的
C. 不一定是频繁的
D. 与频繁性无关
✓ B
解析：先验原理：频繁项集的所有非空子集也必须是频繁的。逆否命题：非频繁项集的超集也一定非频繁（用于剪枝）。
Q3在挖掘关联规则时，支持度阈值的作用是（ ）。单选
A. 保证规则的可靠性
B. 保证规则的相关性
C. 保证规则的可解释性
D. 筛掉出现比较少、没有代表性的项集
✓ D
解析：支持度阈值用于过滤掉出现频率低、无代表性的项集。置信度阈值（如>0.7）用于确保规则可靠性。相关性由提升度等度量评估。
Q4FP-Growth 算法相比于 Apriori 算法的主要优势是（ ）。单选
A. 不产生候选集，只需两次扫描
B. 能处理连续属性
C. 能找到更多频繁项集
D. 不需要最小支持度参数
✓ A
解析：FP-Growth 的核心优势是不产生候选集且只需两次扫描数据库，速度比Apriori快得多。它同样需要最小支持度参数，处理的是离散属性。
Q5Apriori 算法中，L₂ 自连接生成 C₃ 时，要求两个频繁2-项集（ ）。单选
A. 所有项都相同
B. 至少有一个相同项
C. 前 k-2=1 项相同
D. 最后一个项相同
✓ C
解析：自连接规则：L_{k-1} 中的两个项集，若它们的前 k-2 项相同，则可以连接产生 C_k 候选。对于 L₂ 自连接生成 C₃，要求前 1 项相同。
Q6如果关联规则 X→Y 的提升度 lift(X→Y) = 1，则说明（ ）。单选
A. X 和 Y 正相关
B. X 和 Y 相互独立
C. X 和 Y 负相关
D. 规则是强关联的
✓ B
解析：提升度＞1为正相关，＜1为负相关，=1表示独立（X发生不影响Y发生的概率）。独立时不应作为关联规则。
Q7支持度计数 support_count({Milk, Bread, Diaper}) = 2，总事务数为5，则该规则的支持度为（ ）。单选
A. 1/5
B. 2/3
C. 0.5
D. 2/5
✓ D
解析：支持度 = 支持度计数 / 总事务数 = 2/5 = 0.4。注意区分支持度计数（绝对次数）和支持度（比例）。
Q8Apriori 算法中"剪枝步"的作用是（ ）。单选
A. 利用先验性质删除不可能频繁的候选项集
B. 合并相似的频繁项集
C. 减少数据库扫描次数
D. 优化内存使用
✓ A
解析：剪枝步利用先验性质：候选k-项集的任何(k-1)-子集若不在L_{k-1}中，则该候选也不可能是频繁的，直接删除。这减少了不必要的支持度计算。
Q9关联规则挖掘的一般步骤中，第一步是（ ）。单选
A. 生成强关联规则
B. 计算置信度
C. 找出所有频繁项集
D. 扫描数据库
✓ C
解析：关联规则挖掘分两步：①找出所有频繁项集 ②从频繁项集中产生强关联规则。第一步用Apriori/FP-Growth等，第二步计算置信度并筛选。
Q10在FP树构建过程中，项按什么顺序排列？（ ）单选
A. 按字母顺序
B. 按支持度从高到低
C. 按事务出现顺序
D. 随机排序
✓ B
解析：FP树中项按支持度从高到低排列，高频项靠近根节点，这样可以最大化前缀共享，压缩数据。

—— 填空题 ——

FIB_1关联规则挖掘中，筛选频繁项集使用__________阈值，筛选强关联规则使用置信度阈值。填空
支持度（最小支持度）
解析：支持度用于筛选频繁项集（淘汰出现太少的），置信度用于筛选强关联规则（淘汰可信度低的）。
FIB_2Apriori 算法的两个核心步骤是__________和剪枝步。填空
连接步
解析：连接步：L_{k-1}自连接生成C_k；剪枝步：利用先验性质删除不可能频繁的候选。
FIB_3Apriori 先验原理的逆否命题：如果一个项集是非频繁的，则它的所有__________也一定是非频繁的。填空
超集
解析：逆否命题是非频繁项集的所有超集也非频繁，这是Apriori剪枝的数学依据。
FIB_4FP-Growth 算法在构建 FP 树之前，需要先进行__________次数据库扫描找到频繁1-项集。填空
一（1）
解析：FP-Growth 共需两次扫描：第一次找频繁1-项集（确定项头表），第二次构建FP树。相比Apriori每层都扫描，大幅减少IO。
FIB_5提升度 lift(X→Y) = P(X∪Y) / (__________)。当 lift > 1 时表示正相关。填空
P(X) × P(Y)
解析：提升度 = P(X∪Y)/(P(X)P(Y))。>1正相关，<1负相关，=1独立。即"实际共同发生概率/假设独立时应有的概率"。
